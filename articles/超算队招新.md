## Task 1 ：设置一个计算节点，并安装操作系统

我选择了Rocky Linux 10，因为它与Red Hat Enterprise Linux兼容，且与HPC社区习惯相符合。后续需要安装的软件，例如Intel oneAPI HPC Toolkit 2025等，官方支持Rocky。

执行命令`sudo dnf install gcc gcc-c++ gcc-gfortran nfs-utils tmux vim git`安装必要的软件。对于htop，需要先`sudo dnf install epel-release`，然后`sudo dnf install htop`。
我还安装了一些软件：tree（可以很方便地查看文件树结构），unzip/zip/tar（便于解压缩），gdb（便于调试）。

使用命令`sudo hostnamectl set-hostname node1`将主机名设置为`node1`，查看`/etc/hostname`文件，可以看到主机名已经更改。

对于`firewall`和`SELinux`这些服务，集群一般不需要这些服务来保护，而是采用物理隔离和统一管理的方式。这些服务反而可能拦截一些有用的信息，造成一些不必要的麻烦。
关闭防火墙：`sudo systemctl stop firewalld`，`sudo systemctl disable firewalld`。
关闭SELinux：编辑`/etc/selinux/config`文件，将`SELINUX=enforcing`改为`SELINUX=disabled`，然后重启系统。

还有一些服务也用不到，需要关闭，例如bluetooth、cups（打印服务）等。

初始化node2的脚本`init_node.sh`内容如下：

```shell
#!/bin/bash

set -e

HOSTNAME_NEW=$1
hostnamectl set-hostname "$HOSTNAME_NEW"

dnf install -y epel-release
dnf install -y \
    gcc \
    gcc-c++ \
    gcc-gfortran \
    nfs-utils \
    tmux \
    htop \
    git

systemctl stop firewalld
systemctl disable firewalld

sed -i 's/^SELINUX=.*/SELINUX=disabled/' /etc/selinux/config
```

运行命令`chmod +x init_node.sh`给脚本添加执行权限，再运行`sudo ./init_node.sh node2`，即可初始化node2。

## Task 3: 在 `node1` 上增加一个大容量存储设备

我在VMWare的界面上操作，给node1添加了一块80GB的虚拟硬盘。添加硬盘后遇到了启动顺序的问题，进入BIOS，将原有的硬盘的优先级提升到新硬盘之前，然后重启系统。

<img src="./images/scc_recruit/task3/BIOS修改启动顺序.png" width="400"/>

使用`lsblk`命令查看新增的硬盘，使用`sudo fdisk /dev/sda`命令进行分区。

<img src="./images/scc_recruit/task3/硬盘分区.png" width="500"/>

使用`sudo mkfs.ext4 /dev/sda1`命令格式化分区，格式化为ext4文件系统，使用`sudo mount /dev/sda1 /data`命令挂载分区到/data目录下，使用`df -h`命令查看挂载结果，最后一行可以看到`/data`目录已经挂载成功。

<img src="./images/scc_recruit/task3/检查挂载结果.png" width="500"/>

接下来设置开机自动挂载。由于UUID是磁盘分区的唯一标识，且不会发生变化，因此使用UUID来配置开机自动挂载。使用`sudo blkid /dev/sda1`命令查看分区的UUID，使用`sudo vim /etc/fstab`命令编辑fstab文件，在文件末尾添加以下内容：
```
UUID=... /data ext4 defaults 0 0
```
保存并退出，使用`sudo mount -a`命令挂载所有在fstab中定义的文件系统。重启node1后挂载仍然生效。

使用`sudo chmod 777 /data`命令设置/data目录的权限，使得普通用户（如scc用户）可读可写。

## Task 4: 学习 Linux 中的软件环境

我先在Github下载了CMake的源码，解压到`/data/software/source/`目录下，依次执行以下命令：
```
cd cmake-4.3.0-rc1
./bootstrap --prefix=/data/software/install/cmake/4.3.0
make -j2
make install
```

中途遇到了一些问题：发现缺失OpenSSL，采用dnf进行安装；起初使用`-j`，发现编译的并发度过高会导致OOM，内存爆了，因此改为`-j2`。

在配置环境变量之前，在安装目录下可以看到cmake的版本号。

<img src="./images/scc_recruit/task4/cmake安装完毕.png" width="500"/>

接下来配置环境变量PATH，使用`vim ~/.bashrc`命令编辑.bashrc文件，在文件末尾添加以下内容：
```
export PATH=/data/software/install/cmake/4.3.0/bin:${PATH}
```
保存并退出，使用`source ~/.bashrc`命令使配置生效，使用`cmake --version`命令查看cmake的版本。

<img src="./images/scc_recruit/task4/配置好环境变量.png" width="500"/>

把自定义路径放在PATH的最前面和最后面有区别，直接影响系统查找命令的优先级，放在前面的优先级更高，后面的优先级更低。Linux系统在执行命令时，会从左到右依次搜索PATH中的目录，找到第一个匹配的可执行文件就停止搜索。

pkg-config是Linux系统中用于获取已安装库的元信息的工具，通过环境变量`PKG_CONFIG_PATH`可以控制搜索路径。

## Task 5: 安装HPC集群必备软件

首先安装Environment Module模块，使用`sudo dnf install environment-modules`命令安装。

接下来安装GCC的7.x和15.x版本。下载`gcc-15.2.0.tar.gz`，解压后进入`gcc-15.2.0`目录，依次执行以下命令：
```shell
./contrib/download_prerequisites

cd ..
mkdir build-gcc-15.2.0
cd build-gcc-15.2.0

../gcc-15.2.0/configure \
  --prefix=/data/software/install/gcc/15.2.0 \
  --enable-languages=c,c++,fortran \
  --disable-multilib \
  --disable-bootstrap

make -j2
make install
```

7.4.0版本使用同样的方式进行编译安装。

创建`/data/software/modulefiles/gcc`目录，vim编辑15.2.0文件，内容如下：
```Tcl
#%Module1.0

proc ModulesHelp { } {
    puts stderr "GCC 15.2.0"
}

module-whatis "GCC 15.2.0"

prepend-path PATH /data/software/install/gcc/15.2.0/bin
prepend-path LD_LIBRARY_PATH /data/software/install/gcc/15.2.0/lib64
prepend-path MANPATH /data/software/install/gcc/15.2.0/share/man
```

文件7.4.0同理，使用`module use /data/software/modulefiles`命令使之生效，接下来对module进行测试，见下图。
其中，`module use`这个命令只在当前会话生效，如果需要永久生效，需要修改MODULEPATH这个环境变量。

MPI的简单使用方式：
- 编写一个C语言程序，如`mpi_hello.c`，内容如下：
```c
#include <stdio.h>
#include <mpi.h> // 包含MPI库的头文件

int main(int argc, char** argv) {
    int rank, size;
    MPI_Init(&argc, &argv); // 初始化MPI环境
    MPI_Comm_rank(MPI_COMM_WORLD, &rank); // 获取当前进程的rank
    MPI_Comm_size(MPI_COMM_WORLD, &size); // 获取进程总数
    printf("Hello from process %d of %d\n", rank, size);
    MPI_Finalize(); // 结束MPI环境
    return 0;
}
```
- 编译程序，使用`mpicc mpi_hello.c -o mpi_hello`命令编译，使用`mpirun -np 4 ./mpi_hello_`命令运行，其中`-np 4`表示启动4个进程。

mpicc本质上是一个脚本，它在后台调用gcc或者其他编译器，并自动添加MPI所需的头文件路径、库路径和链接选项。`mpicc -show`命令会显示 mpicc 实际传递给底层编译器的完整命令行，下面是示例与解释。

```shell
scc@node1:~/test/mpi$ module load mpich/5.0.0
scc@node1:~/test/mpi$ mpicc -show
# 结果如下：
gcc 
-I/data/software/install/mpich/5.0.0/include   # 头文件路径
-L/data/software/install/mpich/5.0.0/lib       # 库文件路径
-Wl,-rpath -Wl,/data/software/install/mpich/5.0.0/lib  # 运行时库路径
-Wl,--enable-new-dtags                         # 运行时链接器选项
-lmpi                                          # 链接MPI库
```

## Task 6: NFS网络文件系统

在`node1`上依次执行以下命令将`/data`目录导出为NFS：

```shell
sudo systemctl enable --now nfs-server
sudo systemctl status nfs-server
sudo vim /etc/exports
# 在文件末尾添加以下内容：
# /data node2(rw,sync,no_root_squash)
sudo exportfs -arv
```

在`node2`上进行检查：

```shell
scc@node2:~$ showmount -e node1
Export list for node1:
/data node2
```

在`node2`上挂载`node1`的`/data`目录，配置开机自动挂载：

```shell
sudo mkdir /data
sudo vim /etc/fstab
# 在文件末尾添加以下内容：
# node1:/data  /data  nfs  defaults,_netdev  0 0]
mount -a
```

在`node2`上安装Environment Module来使用`node1`上的软件。安装完成后，执行：

```shell
vim ~/.bashrc
# 在文件末尾添加以下内容：
# module use /data/software/modulefiles
source ~/.bashrc
module avail
```

在node2上使用软件：

<img src="./images/scc_recruit/task6/在node2使用软件.png" width="450"/>

如果集群规模很大很大，由于NFS是单点架构，所有节点的I/O操作都需要经过NFS服务器，导致NFS服务器成为性能瓶颈；此外，还存在单点故障的问题，一旦NFS服务器出现故障，整个集群都会受到影响。

大规模集群通常采用分布式文件系统：

| 文件系统 | 特点 |
| --- | --- |
| HDFS | 文件默认复制多份，安全，但不适合HPC低延迟场景 |
| Ceph | 用算法决定存放位置，去中心化，延迟略高 |
| BeegFS | 多个存储位置，统一登记，可横向扩展 |
| Lustre | 超高性能，运维复杂，网络要求高 |

## Task 7: 集群监控系统

我先去了解了一下`node_exporter`，`prometheus`和`Grafana`，以及三者的协同工作方式。`node_exporter`负责收集机器的硬件信息，默认的端口是9100，将信息暴露在`node1:9100/metrics`。`prometheus`使用一个时间序列数据库收集`node_exporter`暴露的指标，默认的端口是9090。`Grafana`负责展示`prometheus`收集到的指标，默认的端口是3000。

接下来进行安装和配置，`node_exporter`需要在`node1`和`node2`上分别安装，`prometheus`和`Grafana`只需要在`node1`上安装即可。这三个工具不需要安装在`/data`目录下，因此首先尝试使用dnf进行安装。

```shell
sudo dnf install node_exporter
sudo systemctl daemon-reload
sudo systemctl enable --now node_exporter

sudo dnf install grafana
sudo systemctl daemon-reload
sudo systemctl enable --now grafana-server
```

`node_exporter`和`Grafana`安装成功，但是`prometheus`安装失败，提示找不到`prometheus`包，因此使用wget下载了prometheus的二进制tar包进行安装。

```shell
# 1. 创建用户 (不创建家目录，不允许登录)
sudo useradd --no-create-home --shell /bin/false prometheus

# 2. 创建配置目录和数据目录
sudo mkdir -p /etc/prometheus
sudo mkdir -p /var/lib/prometheus

# 3. 设置目录权限 (让 prometheus 用户拥有这些目录)
sudo chown prometheus:prometheus /var/lib/prometheus

cd /tmp/prometheus-3.10.0-rc.1.linux-amd64

# 4. 移动 prometheus 到 /usr/local/bin
sudo mv prometheus /usr/local/bin/

# 5. 移动默认配置文件到 /etc/prometheus
sudo mv prometheus.yml /etc/prometheus/

# 6. 修正所有相关文件的属主为 prometheus 用户
sudo chown prometheus:prometheus /usr/local/bin/prometheus
sudo chown prometheus:prometheus /usr/local/bin/promtool
sudo chown prometheus:prometheus /etc/prometheus/prometheus.yml

# 7. 创建 systemd 配置文件
sudo vim /etc/systemd/system/prometheus.service

# 8. 启动 Prometheus并设置开机自启
sudo systemctl daemon-reload
sudo systemctl start prometheus
sudo systemctl enable prometheus
sudo systemctl status prometheus
```

其中 `prometheus.service` 的内容如下：
```
[Unit]
Description=Prometheus Server
Documentation=https://prometheus.io/docs/introduction/overview/
After=network-online.target

[Service]
User=prometheus
Group=prometheus
Type=simple
ExecStart=/usr/local/bin/prometheus \
  --config.file=/etc/prometheus/prometheus.yml \
  --storage.tsdb.path=/var/lib/prometheus/ \
  --web.console.templates=/etc/prometheus/consoles \
  --web.console.libraries=/etc/prometheus/console_libraries \
  --web.listen-address=0.0.0.0:9091 \
  --web.enable-lifecycle

Restart=always

[Install]
WantedBy=multi-user.target
```

再修改`prometheus.yml`文件，在`scrape_configs`下添加`node1`和`node2`的配置：
```yml
scrape_configs:
  - job_name: 'node_exporter'
    static_configs:
      - targets: ['node1:9100', 'node2:9100']
```
